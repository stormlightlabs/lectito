About Lectito

Lectito is a free, open-source tool for extracting readable content from web pages. Named after the Latin word for "reading," it strips away clutter—ads, navigation, tracking scripts—and delivers clean, well-formatted articles in your choice of Markdown, HTML, plain text, or JSON.

Built with Rust for performance and reliability, Lectito uses sophisticated content extraction algorithms to identify the main article text on any web page. It handles everything from modern JavaScript-heavy sites to legacy table-based layouts, extracting clean content in under three seconds.

The service is completely free for personal use, with generous rate limits that accommodate most research and automation needs. For developers, we offer a simple REST API that integrates seamlessly into workflows, research tools, and content pipelines.

Features

Smart Content Extraction

Advanced algorithms detect and extract article content while removing ads, navigation, sidebars, and other clutter.

Multiple Output Formats

Export as Markdown, HTML, plain text, or structured JSON with full metadata.

Intelligent Caching

Frequently requested URLs are cached for 24 hours, improving response times and reducing load.

Open Source

Built with Rust and Svelte. Self-host or contribute on GitHub under the MIT License.

Rate Limits

To ensure fair usage and service stability, the API enforces rate limits per IP address across three time windows:

Window Limit Reset
Per Minute 60 requests Every minute
Per Hour 600 requests Every hour
Per Day 5,000 requests Daily (UTC)

Tip: Check your current rate limit status at any time by calling GET /api/v1/limits. All responses include rate limit headers.

API Reference

POST /api/v1/extract

Extract article content from a URL. This is the primary endpoint for content extraction.

Request Body

{
  "url": "https://example.com/article",
  "format": "markdown",
  "include_frontmatter": true,
  "include_references": false,
  "strip_images": false,
  "content_selector": null
}

Response

{
  "content": "# Article Title\n\nArticle body content...",
  "metadata": {
    "title": "Article Title",
    "author": "Jane Doe",
    "date": "2026-01-15",
    "excerpt": "Brief description...",
    "site_name": "Example Site",
    "language": "en",
    "word_count": 2450,
    "reading_time_minutes": 12.5,
    "image": "https://example.com/image.jpg",
    "favicon": "https://example.com/favicon.ico"
  },
  "cached": true,
  "extracted_at": "2026-04-05T12:00:00Z"
}
GET /api/v1/extract

GET variant for simple integrations and browser use. Parameters are passed as query strings.

Query Parameters

GET /api/v1/extract?url=https%3A%2F%2Fexample.com%2Farticle&format=markdown
GET /api/v1/library

Browse cached and previously extracted articles. Returns paginated metadata without full content.

Query Parameters

  • page — Page number (default: 1)
  • per_page — Items per page (default: 20, max: 100)
  • sort — recent, popular, or alpha (default: recent)
  • q — Search query for title/domain (optional)
GET /api/v1/health

Health check endpoint. Returns service status and version information.

{
  "status": "ok",
  "version": "1.0.0"
}
GET /api/v1/limits

Returns current rate limit status for the calling IP address.

{
  "requests_remaining": 55,
  "requests_limit": 60,
  "window_seconds": 60,
  "reset_at": "2026-04-05T12:01:00Z"
}

Example Usage

cURL

curl -X POST https://api.lectito.io/api/v1/extract \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com/article", "format": "markdown"}'

JavaScript

const response = await fetch('https://api.lectito.io/api/v1/extract', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    url: 'https://example.com/article',
    format: 'markdown'
  })
});

const data = await response.json();
console.log(data.content);

Error Codes

Code Meaning Description
400 Bad Request Invalid URL or malformed request body
403 Forbidden URL blocked by spam filter or domain blocklist
429 Too Many Requests Rate limit exceeded. Retry-After header provided.
502 Bad Gateway Failed to fetch the requested URL
504 Gateway Timeout Upstream fetch timed out (30s limit)
